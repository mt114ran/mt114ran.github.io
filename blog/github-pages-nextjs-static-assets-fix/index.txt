13:T1862,<h1 id="github-pagesでnextjsの静的アセットが404になる問題の解決方法">GitHub PagesでNext.jsの静的アセットが404になる問題の解決方法</h1>
<p>Next.jsでビルドした静的サイトをGitHub Pagesにデプロイした際、CSSやJavaScriptファイルが404エラーになり、スタイルが適用されない問題に遭遇しました。本記事では、この問題の原因と解決方法について説明します。</p>
<h2 id="問題の症状">問題の症状</h2>
<ul>
<li>ローカル環境では正常に表示される</li>
<li>GitHub Pagesにデプロイすると、Tailwind CSSのスタイルが全く適用されない</li>
<li>ブラウザの開発者ツールを確認すると、<code>/_next/static/</code>以下のCSSやJSファイルが404エラーになっている</li>
</ul>
<h2 id="原因">原因</h2>
<p>GitHub Pagesは、デフォルトでJekyllを使用してサイトを処理します。Jekyllには以下の特徴があります：</p>
<ol>
<li><strong>アンダースコアで始まるディレクトリを無視する</strong>：<code>_next</code>や<code>_app</code>のようなディレクトリは、Jekyllによって無視され、公開されません</li>
<li><strong>静的ファイルの処理</strong>：Jekyllは、サイトを再処理してビルドを行うため、意図しない変更が加わる可能性があります</li>
</ol>
<p>Next.jsの<code>next export</code>（静的エクスポート）では、<code>_next</code>ディレクトリにすべてのJavaScriptとCSSファイルが出力されるため、このディレクトリがJekyllによって無視されることで、アセットファイルにアクセスできなくなります。</p>
<h2 id="解決方法">解決方法</h2>
<h3 id="1-nojekyllファイルの追加">1. <code>.nojekyll</code>ファイルの追加</h3>
<p>プロジェクトのビルド出力ディレクトリ（通常は<code>out</code>）に<code>.nojekyll</code>という空のファイルを追加します。このファイルは、GitHub PagesにJekyllの処理をスキップするよう指示します。</p>
<pre><code class="language-bash">touch out/.nojekyll
</code></pre>
<h3 id="2-ビルドスクリプトの修正">2. ビルドスクリプトの修正</h3>
<p><code>package.json</code>のビルドスクリプトを修正して、ビルド後に自動的に<code>.nojekyll</code>ファイルを作成するようにします：</p>
<pre><code class="language-json">{
  "scripts": {
    "build": "next build &#x26;&#x26; touch out/.nojekyll",
    "deploy": "gh-pages -d out -t true"
  }
}
</code></pre>
<p>重要なポイント：</p>
<ul>
<li><code>touch out/.nojekyll</code>：ビルド後に<code>.nojekyll</code>ファイルを作成</li>
<li><code>gh-pages -d out -t true</code>：<code>-t</code>オプションを追加して、ドットファイル（<code>.nojekyll</code>）も含めてデプロイ</li>
</ul>
<h3 id="3-github-actionsを使用する場合">3. GitHub Actionsを使用する場合</h3>
<p>GitHub Actionsを使用してデプロイする場合は、以下のようなワークフローを設定します：</p>
<pre><code class="language-yaml">name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
          
      - name: Install dependencies
        run: npm ci
        
      - name: Build
        run: npm run build
        
      - name: Add .nojekyll
        run: touch out/.nojekyll
        
      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./out
</code></pre>
<h2 id="確認方法">確認方法</h2>
<p>デプロイ後、以下の点を確認します：</p>
<ol>
<li><strong>アセットファイルへのアクセス</strong>：<code>https://[username].github.io/[repository]/_next/static/</code>にアクセスして、ファイルが正しく配信されているか確認</li>
<li><strong>開発者ツールの確認</strong>：NetworkタブでCSSやJSファイルが200 OKで読み込まれているか確認</li>
<li><strong>スタイルの適用</strong>：ページが正しくスタイリングされているか視覚的に確認</li>
</ol>
<h2 id="トラブルシューティング">トラブルシューティング</h2>
<h3 id="問題が解決しない場合">問題が解決しない場合</h3>
<ol>
<li><strong>キャッシュのクリア</strong>：ブラウザのキャッシュをクリアして再読み込み</li>
<li><strong>GitHub Pagesの再デプロイ</strong>：GitHubのリポジトリ設定でGitHub Pagesを一度無効にして、再度有効化</li>
<li><strong>ビルド出力の確認</strong>：<code>out</code>ディレクトリに<code>.nojekyll</code>ファイルが存在することを確認</li>
</ol>
<h3 id="nextjsの設定確認">Next.jsの設定確認</h3>
<p><code>next.config.js</code>（または<code>next.config.ts</code>）の設定も確認します：</p>
<pre><code class="language-javascript">const nextConfig = {
  output: 'export',
  trailingSlash: true,
  // GitHub Pagesで公開する場合、適切なbasePathを設定
  basePath: process.env.NODE_ENV === 'production' ? '/repository-name' : '',
  assetPrefix: process.env.NODE_ENV === 'production' ? '/repository-name' : '',
}
</code></pre>
<p>ただし、ユーザー名と同じ名前のリポジトリ（<code>[username].github.io</code>）の場合は、<code>basePath</code>と<code>assetPrefix</code>は空文字列で問題ありません。</p>
<h2 id="まとめ">まとめ</h2>
<p>GitHub PagesでNext.jsの静的サイトをホスティングする際の404エラーは、Jekyllがアンダースコアで始まるディレクトリを無視することが原因です。<code>.nojekyll</code>ファイルを追加することで、この問題を簡単に解決できます。</p>
