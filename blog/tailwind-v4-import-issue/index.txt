13:T10dc,<h1 id="tailwind-css-v4のimport構文でスタイルが適用されない問題と解決方法">Tailwind CSS v4の@import構文でスタイルが適用されない問題と解決方法</h1>
<p>Next.js 15.3.3とTailwind CSS v4を使用していて、スタイルが正しく適用されない問題に遭遇しました。この記事では、その原因と解決方法について説明します。</p>
<h2 id="問題の症状">問題の症状</h2>
<p>以下のような構成でTailwindのスタイルが全く適用されませんでした：</p>
<pre><code class="language-css">/* globals.css - 問題のあるコード */
@import "tailwindcss";

:root {
  --background: #ffffff;
  --foreground: #171717;
}

body {
  background: var(--background);
  color: var(--foreground);
}
</code></pre>
<h2 id="原因">原因</h2>
<p>Tailwind CSS v4では新しい<code>@import</code>構文が導入されましたが、Next.jsのビルドプロセスとの互換性に問題がある場合があります。特に静的エクスポート（<code>output: 'export'</code>）を使用している場合、この新しい構文が正しく処理されないことがあります。</p>
<h2 id="解決方法">解決方法</h2>
<p>従来の<code>@tailwind</code>ディレクティブを使用することで問題を解決できます：</p>
<pre><code class="language-css">/* globals.css - 修正後 */
@tailwind base;
@tailwind components;
@tailwind utilities;

:root {
  --background: #000000;
  --foreground: #ffffff;
}

html {
  color-scheme: dark;
}

body {
  @apply bg-black text-white;
  font-family: Arial, Helvetica, sans-serif;
}
</code></pre>
<h2 id="ダークテーマの実装">ダークテーマの実装</h2>
<p>今回は背景を黒、テキストを白にするダークテーマも同時に実装しました。</p>
<h3 id="1-グローバルスタイルの設定">1. グローバルスタイルの設定</h3>
<pre><code class="language-css">/* ダークモードを強制適用 */
html {
  color-scheme: dark;
}

body {
  @apply bg-black text-white;
}
</code></pre>
<h3 id="2-コンポーネントのカラー調整">2. コンポーネントのカラー調整</h3>
<p>各コンポーネントで使用している色も調整が必要です：</p>
<pre><code class="language-tsx">// リンクの色を調整
&#x3C;Link href="/blog" className="text-blue-400 hover:text-blue-300 hover:underline">
  記事一覧へ
&#x3C;/Link>

// グレーテキストの調整

  作成日: {post.create}
&#x3C;/p>

// ボーダーの色調整
&#x3C;li className="border-b border-gray-700 pb-4">
  {/* コンテンツ */}
&#x3C;/li>
</code></pre>
<h2 id="ベストプラクティス">ベストプラクティス</h2>
<h3 id="1-tailwind設定の確認">1. Tailwind設定の確認</h3>
<p><code>tailwind.config.ts</code>でコンテンツパスが正しく設定されているか確認：</p>
<pre><code class="language-typescript">const config: Config = {
  content: ['./src/**/*.{js,ts,jsx,tsx,mdx}'],
  theme: {
    extend: {},
  },
  plugins: [],
}
</code></pre>
<h3 id="2-postcssの設定">2. PostCSSの設定</h3>
<p><code>postcss.config.mjs</code>でTailwindが正しく設定されているか確認：</p>
<pre><code class="language-javascript">export default {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
  },
}
</code></pre>
<h3 id="3-ビルドの確認">3. ビルドの確認</h3>
<p>変更後は必ずビルドを実行して、エラーがないことを確認：</p>
<pre><code class="language-bash">npm run build
</code></pre>
<h2 id="まとめ">まとめ</h2>
<p>Tailwind CSS v4の新しい<code>@import</code>構文は、まだすべての環境で完全にサポートされていない可能性があります。問題が発生した場合は、従来の<code>@tailwind</code>ディレクティブを使用することで解決できます。</p>
<p>また、ダークテーマを実装する際は、以下の点に注意が必要です：</p>
<ol>
<li>グローバルスタイルで基本的な色を定義</li>
<li>各コンポーネントで使用している色クラスを調整</li>
<li>ボーダーや背景色も忘れずに調整</li>
</ol>
